Week 1

Software System Life Cycle

Software System Life Cycle (SSLC) - 5 stage development of projects and are cyclical in nature not always from end to end. Analysis - questions asked and answered to establish goals and needs of the project. Design - Design for project determined using one or more approaches of topdown/bottom up/ OOD. Implementation - code is written Testing - test are run against the code to determine if it works properly Maintenance - After the project is out, these are adjustments that have to be made after release.

Software Design Strategies

Top down - breaking process down into smaller peices which is also known as step-wise reinforcement. Bottom Up - using existing libraries and APIs. Object Oriented Design - abstraction, inheritance, polymorphism.

Visual Organizers

Flowchart - Shapes to signify different programming constructs. Diamond is an if statement, using process blocks. Unified MOdeling Language(UML) - used to visualize an object oriented design system.

Software Development Models

Waterfall - commonly used in industry production during early years. Progress flows steadily down through the phases of idea, analysis, development, test. This model was rigid and does not easily allow for modifications. Pros - straightforward, easy Cons - changes can be costly Incremental - small steps completing all software system life cycle steps for each subset then repeats the cycle for each portion until the project is complete. Pros - flexible Cons - requires good planning and design to work Agile Methodologies(SCRUM,XP, Kanban) - Agile refers to a set of four principals and are both incremental and iterative. They emphasize continuous delivery of software that is coded, tested, and fixed in 1-3 weeks durations. Pros - flexible, can reduce bugs Cons - prone to feature creep, weak documentation Spiral - risk driven process model. This model guides the team to a specific process based on the risk analysis of the problem. Pros - flexible Cons - complex, difficult documentation, costly.

Style Conventions

Spacing and indentation - Python and some other languages require specific spacing and indentation. Other languages do not require it but it is important for readability and maintenance. Descriptive Identifiers - names should accurately describe the data or process. Case-Specific Identifiers - class names are named using pascal camelcase like CollegeStudent, for variables they use lowercase camel case like firstName getFirstName. Constants are usually in all CAPS like PI. Comments for Documentation - comments are embedded in the program and ignored by the compiler. Pre-and-Post-Condition-Comments - pre condition is what must be true to start the task, postconditon is what must be true to be complete.

Robust Program Creation

Robust - ability to overcome adverse conditions, handling negative issues gracefully. Unit Testing - testing each individual part to make sure it works in all conditions. Input Protection and Data Validation - making sure data is entered in the proper ways. Exception - handling errors that is thrown when something bad happens. Things like IndexOutOfBoundsException or FileNotFoundException. Exception Handling - the process of handling error messages like try catch block.

Introduction to Object Oriented Programming

Object Oriented Programming - programming language model using objects containing data as the focus. Imperative (Procedural) Programming - when the code describes in exacting detail the steps the computer needs to take. Function Programming - pure functional approach to problem solving. Object - instance of a class that encapsulates data and related actions or processes called methods. Class - definition of an object - type of data owned by the object as well as the methods. Encapsulation - incorporates data items and processes into a class definition instance variables, constructors accessors and modifier methods. Abstraction - programmer hides all but the relevant data needed to interact with the object.

Inheritance and Composition

Inheritance - allows classes to be defined based on previous classes receiving all the parent attributes. "is a " - the class is derived from another class like a student is a person. Child Class - inherits the characteristics of the parent class. Class - definition of an object describing the data owned by the object and the methods. Composition - combining simpler objects into more complex ones. A car may have a body type object, an engine, transmission, and is composed of all these different classes. "has a " - one class is composed of simpler classes such as hte car and engine objects.

Polymorphism

Polymorphism - means many forms where methods or processes belong to the object can be overridden or replicated as needed. Overloading - constructors and methods are named the same but operate on different input parameters. Overriding - when inherited characteristics are redefined and customized to fit that object from the parent. Polymorphic Objects - refers to how parents can reference children, several levels of inheritance down.

Data Types

3 general types of data and several specific types. Data - Numbers, text, boolean. Data types - Integer (int or short) negative or non negative number. Short usually uses half hte memory. Floating point number (float or double) - real number with fractional parts double uses twice the storage of float and has more precision than float. Character (char) - single symbol, letter, digit, white space, with a pair of single quotes. String - row of characters combined to make a word or group of words contained in double quotes. Boolean - true or false values from Boolean Algebra which logic processses originate from. Data types Storage and limitations integers - int, short, long, byte. byte uses 8 bits of storage ranging from -128 to 127. short uses 16 bits of storage ranging from -32768 to 32767. int 32 bits of storage -2,147,483,648 to 2,147,483,647. The long data type is 64 bits +- 9 quintillion. The char data stores characters a cousin of integer and stores integers as characters. It uses 16 bits of storage and contains only non negative in the range 0 - 65535,, the most significant values are 65, 97, 48. Uppercase 'A' is 65, 'a' is 97, '0' is the value 48. Wraparound issues - wrap around occurs when the variable contains the maximum or minum value then tries to store one more or one less beyond the value. Like 127 + 1 is -128 for a byte.

variable - location in memory containing a value it can be changed during program execution unlike a constant. variable/constant-first declared request to allocate memory,instantiated or assigned a value. Ex. int x<-5 valid identifiers - only alphanumeric + _ . Must begin with alpha or underscore. No symbols, no spaces, no reserved. conventions - "best practice" some identifiers start with an upper case letter such as class names. procedures, functions, methods start with a lower case letter. for multiple words camelCase is used where each new word starts with a capitalized letter. constants are completely uppercased. <-, = used to assign variables and constants. == check for equality between expressions != used to check if expressions are not equal to each other. += add or concatenate the new value to the previous assigned -= subtract the new value from the previous value assigned *= multiply the new value by the previous value assigned /= divide the previous value by the new value assigned

Primitives vs Objects

Primitives - basic in form, raw. Data types such as integer, character, float, boolean are considered to be primitive. They occupy memory that is consistent in size. All primitives are stored and accessed directly in their memory. Objects - are not consistent in how much memory they occupy. The object itself cannot be stored and accessed directly. Objects are stored by reference through use of pointers, the variable stores the memory address. Objects include stringes, one dimensional and 2 dimensional arrays. Primitives vs objects - in many cases there is no real difference, but when the complexity gets greater the difference will be important to know.

What is Sequential Processing

In sequential processing statements execute in order one after another. One step cant begin until the previous has completed.

Debugging Strategies

Bug/error - error flaw or failure causing incorrect or unexpected results. Lexical Errors - error characterized by misspelling a word, capitals, words out of order. Syntax error - error associated with missing punctuation, incorrect indents, mismanaged symbols Run-time errors - error that occurs during execution of the program. Logic error - causes incorrect functionality but the program still runs. Compile time errors fall under lexical errors or syntax. Run time / logic errors occur at runtime.
