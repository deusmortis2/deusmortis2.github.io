  • When should we use arrays?
    If we were making a video game of our own and we wanted to store top player scores. Instead of storing each score in a variable, we could use an array for topscores. You can assign a value to an array after initializing like int scores[5]; we can use the index like scores[0] = 120; if you know all 5 ahead of time you can do int scores[5] = {120, 86, 56, 42, 33}; In C an array does not have a dynamic length it's static from initialization.
  • What are strings, really?
    Not native to C, it is a part of the CS50 library. A string is basically just an array of characters. Strings always end in a null value or \0 which is the marker for the ending of the string.     Arguments are used to tell the command more details like what to do the command on or where to go to. It can also just gather inputs by using the int main(int argc, string argv[]) main start. Command line arguments may save time in comparison to using get inputs. make mario is gonna assign argv "make" and "mario" with argc = 2.
  • What makes for good design?
    Is your code efficient? Avoid unnecessary repetition and redundant operatios. Avoid copy/paste, if something keeps being repeated find a better way. Is your code easy to maintain? Such as using prototyping, and avoiding magic numbers.

  • How does recursion work?
    Recursion works by solving a problem by solving smaller versions of the same problem. Every recursive function should have: base case - when to stop running the function, and a recursive case - a call to teh function to solve a smaller version of the problem. In a factorial program the base case is 1, and the recursive case its calling the number times the function called with the num - 1.
  • How can we search and sort efficiently and What is Big O Notation?
    If we search linearly for a specific value, it would take n number of times to check each one potentially. It could also be as little as 1 time. O(n) = "scaling" definition for every item that gets added to the input the algorithm will need to do that many searchs. Selection sort works like : Repeat for every element in our list, except last: | Assume the element at this position is the smallest for every element to teh right: | if less than our current smallest: | update the current smallest | swap the current smallest with the current element. Pseudocode like : For i from 0 to n - 2 | Set min = i | for j from i + 1 to n - 1 | if array[j] < array[min] | update min = j | swap array[i] with array[min]. O(n^2) is the upper bound definition. The best case is still n^2 there is no way to make the search shorter. Bubble sort - move through each set of 2 and swap if they are out of place. After each pass the last digit will be in the right place. Repeat n - 1 times | set swaps to none | for j from 0 to n - 2 | if jth and j + 1th element out of order | swap them | if swaps = none | quit. Upper bound is still O(n^2) but the lower bound is just n. Merge sort - basically splits the lists until there is only one value in the list, and then progressively merges the lists together to create a sorted list comparing values to each other value. Merge pseudocode - Split the list into two halves | Sort the left half using merge sort | Sort the right half using merge sort | Merge the two halves into a single sorted list: | take the smaller first element from either half. Upper bound is O (n log n), the omega is n log n, making it have a theta too.
